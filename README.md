# Friday Local AI Assistant

Friday is a fully local, privacy-focused AI assistant built with Python. It listens to your voice commands, processes them using a local Llama 3.2 model via Ollama, and responds with synthetic speech generated by Piper TTS. Friday can also interact with your local Windows operating system to perform tasks, such as opening applications.

## Features

- **Local Speech-to-Text (STT):** Uses Faster-Whisper to quickly transcribe your voice securely on your machine.
- **Local Large Language Model (LLM):** Powered by Ollama running Llama 3.2 for intelligent and private conversational capabilities.
- **Local Text-to-Speech (TTS):** Uses Piper TTS to provide fast, high-quality voice responses.
- **Action Engine:** Can dynamically execute system commands on your Windows machine based on your voice requests.
- **Asynchronous Architecture:** Built using Python's `asyncio` for non-blocking, smooth interactions.

## Prerequisites

Before running the assistant, ensure you have the following installed or set up:

1. **Python 3.8+**
2. **Ollama:** 
   - Download and install [Ollama](https://ollama.ai/).
   - Once installed, open your terminal and run the following command to download the Llama 3.2 model:
     ```bash
     ollama run llama3.2
     ```
3. **Microphone Setup:** Ensure your microphone is properly connected and functioning in Windows.

## Installation

1. **Navigate to the Project Directory:**
   ```bash
   cd friday
   ```

2. **Install Python Dependencies:**
   ```bash
   pip install -r requirements.txt
   ```
   *Note: This installs `faster-whisper`, `SpeechRecognition`, `pyaudio`, `aiohttp`, and `piper-tts`.*

3. **Piper TTS Setup:**
   The project requires a Piper TTS binary (`piper.exe`) and an ONNX model to generate speech.
   - The necessary files like `piper.exe` and `en_GB-jenny_dioco-medium.onnx` should be present in the project directory. If you want to use a different voice, download a Piper ONNX model and its corresponding `.json` file.
   - Ensure the `.onnx` model files are located in the project's root folder or the `models/` directory so the script can locate them.

## Running the Assistant

1. **Start Ollama:** Ensure Ollama is running in the background. The assistant communicates with Ollama locally over `http://localhost:11434`.

2. **Run the Application:**
   Open a terminal in the project directory and execute:
   ```bash
   python main.py
   ```

3. **Interact with Friday:**
   - Wait for the console to output `[System] Friday is now active. Press Ctrl+C in the terminal to exit.`
   - Wait for the prompt `[Mic] Listening... (Speak now!)` before talking.
   - **Example Conversational Command:** "Friday, what is the capital of France?"
   - **Example Action Commands:** "Friday, open the terminal" or "Friday, open notepad".
   - Press `Ctrl+C` in the terminal to safely shut down the assistant.

## Architecture

The system operates via four primary asynchronous queues:
- **The Ears (`listen_for_audio`):** `SpeechRecognition` grabs audio from the microphone, and `Faster-Whisper` transcribes it.
- **The Brain (`process_llm`):** The transcribed text is sent to the local Ollama Llama 3.2 API. It routes the output to either the Action Queue (if formatted as a JSON system command) or the TTS Queue (if standard conversational text).
- **The Hands (`execute_action`):** Processes JSON payloads and runs commands locally via the OS shell.
- **The Mouth (`speak_tts`):** Takes conversational text, synthesizes it to a temporary `.wav` file using Piper, and plays it back using the native Windows `winsound` library.
